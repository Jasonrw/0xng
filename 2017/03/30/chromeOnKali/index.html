Kali环境下安装google-chrome

Kali
  • Kali
  • Web Browser
  • chrome
  • shadowsocks

遇到的问题

用习惯了Chrome，切换到Kali时，默认的Firfox用不习惯，于是想装上Chrome（主要是Google Account里面保存了各种账户，习惯了不用填密码）

摸索

要装最新版的Chrome，就需要上Google官网上去下载，肯定要先解决FQ的问题。

换了kali的源

阿里云的源：

安装shadowsocks

两种方式:

  1. 通过python的pip来安装
    pip install shadowsocks
  2. 直接通过包管理器安装
    apt-get install shadowsocks

安装完成后，配置好/etc/shadowsokcs/config.json，启动ss
sslocal -c /etc/shadowsocks/config.json
将系统代理设置成127.0.0.1:1080，发现并不能上网，通过shell执行
curl -x 127.0.0.1:1080 http://google.com
提示错误，代码500，明显是代理有问题，重新开启sslocal并显示详细信息
sslocal -c /etc/shadowsocks/config.json -vv
发现一旦使用代理企图连接互联网就会不停的报
sweeping timeout
找到GitHub上托管的shadowsocks项目中的issues，果然发现有人遇到了和我一样的问题，有人发现是因为DNS并没有通过ss所以导致无法解析墙外的网站如FB，Google等等，并提示可以通过修改/etc/resov.config来修改DNS，修改完后重启网络
/etc/init.d/networking restart
/etc/init.d/network-manager restart
发现还是不行，突然发现resolv文件第一句话是generate by network-manager，于是直接在右上角的wifi管理中修改我的wifi的IPV4的DNS服务器（8.8.8.8），并关掉自动选项，重启计算机后代理可以正常FQ

安装Chrome

直接上Google官网下载Chrome for Debian,下载好后通过
dpkg -i Google-*
安装，提示依赖有问题（缺少依赖），执行
apt-get -f -y install
自动安装好了缺少的依赖，重新执行
dpkg -i Google-*
提示google-chrome安装成功，启动Google-Chrome
google-chrome
提示已放弃,启动不成功，想到Chrome一直都不允许在Root权限下运行，上网看了很多相关的解决方案，最后用命令
google-chrome --user-data-dir --no-sandbox
启动成功，但是进去以后提示--no-sandbox是不安全的（实际上，在root权限下使用任何图形化界面的软件都是不安全的），本身是搞安全的，对这方面比较敏感，决定还是不在Root权限下运行了，于是新建了一个用户
adduser chromeuser
建好后用gksu来指定用chomeuser运行chrome
gksu -u chromeuser google-chrome
提示在/run/0/目录下建立目录的权限不够，虽然也能打开，但是看着挺难受的，于是修改了改目录的权限
chmod 777 /run/0/
再次运行
gksu -u chromeuser google-chrome
后终于见到了正常的Chrome界面（当场热泪盈眶有木有啊…），但是浏览器是经常要用的，总不能每次打开都来这么一句命令啊，于是打开kali的设置–>键盘–>快捷键
添加了一个快捷键
ctrl + alt + w
其对应的命令就是gksu -u chromeuser google-chrome，终于可以快捷键打开Chrome了

总结

在折腾Chrome的过程中学到了很多知识，总结如下：

  1. 生命在于折腾！
  2. 解决ss开启后提示”sweeping timeout”
  3. 就算开启了ss和linux的全局代理，DNS也不会走ss去解析请求的域名
  4. 通过gksu来用其他用户的权限运行程序
  5. 遇到问题要上项目官网的issues上找“同病相怜”的哥们
