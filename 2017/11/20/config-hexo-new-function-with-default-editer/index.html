配置hexo new时自动打开默认的编辑器

hexo
  • hexo

问题

突然感觉之前写博客的时候，先输入
hexo new post-title
然后通过remarkable打开
remarkable source/_posts/post-title.md
的方式很傻，于是上Google上找找有没有自动的解决方案。结果直接被各种我也不知道是不是不负责任的博客给坑了。

探索

Google上有好几种方法，看起来都比较类似，都是在hexo根目录下新建一个叫做scripts的文件夹，然后在里面随便新建一个文件，把类似以下的代码填进去

但是大部分都是只提到了Win或者Mac的使用方法，我等大Ubuntu党们情何以堪。
其中又分成2.x的版本和3.x的版本，我查了以下我自己的版本

选用了3.x的版本试了一下，发现不行，提示找不到function exec。自己看了看js的代码（我不会js），发现spawn这个变量居然没有用，感觉定义了不用，作者肯定不会这么笨（肯定比我聪明多了。笑～）。于是用spawn变量代替exec。发现果然可以了

结果

Ubuntu下hexo 3.x的版本我来补充一下，需要使用自己编辑器的可以手动修改/usr/bin/remarkable

此致。
