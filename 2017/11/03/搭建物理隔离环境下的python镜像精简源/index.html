搭建物理隔离环境下的python镜像精简源

python
  • pip
  • pip2pi
  • python
  • 环境搭建

问题

工作的环境是个物理隔离的环境，平时开发都是用的QT大礼包（盛情推荐！！），但是在家用惯了python的我还是感觉很不爽，于是起了心思在该环境中搭建一个python源，于是开始了各种刷知乎刷微薄刷票圈…额，不是，是刷Google。

探索

1. bandersnatch

先是想Clone整个pip源，毕竟一劳永逸，以后再也不用考虑库的事，找到了一个神器bandersnatch，python官方发布，直接可以把整个源拉下来，正在高兴的看着怎么用这玩意的时候发现了这句话

It will populate your mirror with the current status of all PyPI packages - roughly 500GiB (2017-02-12). Expect this to grow substantially over time.

马上就停了下来，由于物理隔离的环境，硬盘是肯定插不了的，只能刻光盘，按照4.7G的量来算，至少需要125张光盘…直接否了。

2. pip2pi

pip2pi也是个官方的用于下载python库的小工具，可以按照列表来下载包并搭建源，看到这个我感觉这就是我要找的东西，但是列表从哪里来呢？这是个问题。

1. 知乎

上知乎找常用的包有哪些，哪些包比较好用，确实有一些列表，但是我感觉完全不能覆盖日常的所有需求，至少有好多我平时用的都没覆盖到，再加上格式上不统一，无法用爬虫来爬取页面自动生成列表文件，再次否了。

2. Google

鉴于python有个好用的pip工具，很少有人会发布常用的包系列类似的文章，更没有人会去打包常用库给大家下载（估计除了我也没有别人会去下载了），Google也没找到好东西。

3. 自己统计

实在找不到现成的轮子了，只好自己造了。于是开始找pip一年的下载记录，然后准备将他排序，取前n个软件，用pip2pi来下载。

结果

最后找到了这个玩意，有人统计了一年python各个版本的库的下载量

Python 2.7 still reigns supreme in pip installs

然后发现了一个google的大数据平台的一个库（数据库），有很长一段时间pip的各种下载记录

PSF downloads table

通过查询，得到了一个一年以来python2.7的库的下载量的desc排序表，取得大于10W的库的名称，输入到一个叫做requirements.txt的文件里面。通过pip2pi来批量下载。你以为这样就结束了？那也太难易五了。

解决pip2pi下载失败的问题

本来以为这就结束了的，突然发现pip2tgz这个命令在下载过程中会各种花式结束进程，原因是在本机装不上这个要下载的包（不太明白为什么）。基本上下载个<10个包就会停止一次，需要手动重新启动，刚开始我还认真的手动启动，后来实在忍不了了。写了个小程序来下载，核心代码如下

void threadPip::run(){
	QFile file("C:\\Users\\jason\\req1.txt");
	file.open(QIODevice::ReadOnly);
	int count = 0;
	while(!file.atEnd()){
		QString line = file.readLine();
		qDebug()<<line;
		QProcess pip2tgz;
		pip2tgz.start("C:\\Users\\jason\\AppData\\Local\\Programs\\Python\\Python35\\Scripts\\pip2tgz.exe", QStringList()<<"C:\\Users\\jason\\pip3\\"<<line.trimmed());
		pip2tgz.waitForFinished(3000000);
		count++;
		//ui.label->setText(QString::number(count));
		//ui.progressBar->setValue((double)count/596.0);
		qDebug()<<count;
		emit cur_count(count);
	}
}

做个线程池，然后多线程自己下载，下载失败了的直接跳过。睡了一觉醒来，发现下载了1000+个包，略微浏览了一遍，发现常用的都下载结束了，总共不到1G，高高兴兴的刻盘，搞定！

附 python2.7 年下载量超过10W的包

需要的可以自行拷贝回去用。
